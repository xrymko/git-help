Импорт репозитория из GitHub в локальную папку

Переходим в папку проекта

1 способ2 способ
Через проводник заходим в папку проекта, ПКМ -> открыть GIT BASH

Команда cd в терминале

cd путь/к/папке

подняться на 1 директорию выше

cd ..

Копируем url репозитория

Далее заходим в GIT BASH и пишем git clone и добавляем ссылку. После ссылки без пробела ставим .git

Итого

$ git clone https://xrymko.github.io/presentations.git

Проект будет находится в папке, которую вы выбрали

Экспорт с локальной папки в репозиторий GitHub

Переходим в папку проекта

1 способ2 способ
Через проводник заходим в папку проекта, ПКМ -> открыть GIT BASH

Команда cd в терминале

cd путь/к/папке

подняться на 1 директорию выше

cd ..

Инициализием репозиторий

$ git init

Подключаем удаленный репозиторий

git remote add origin https://github.com/ваш-логин/ваш-репозиторий.git

Добавляем все файлы в коммит

git add .

Добавляем коммит

git commit -m "имя-коммита"

Предавайте большое значение названиям, чтобы вам было проще ориентироваться в изменениях проекта

Перед тем как залить проект на гид, нужно проверить на какой ветке вы находитесь. Её название находится справа (main). Так же узнать ветку можно с помощью команды (возле неё будет знак *):

git branch

Если требуется изменить ветку (например локальная ветка не соответсвует удалённой или нужна новая ветка)

git branch -m master main

master - название текущей ветки, main - новое название

Осталось загрузить проект

git push origin master

Вместо master пишем свою ветку

Возможные ошибки

remote origin already exists

это значит, что удаленный репозиторий уже подключен

Проверяем привязанные репозитории

git remote -v

Если подключение есть + оно не нужное, можно удалить подключение:

git remote remove origin

Не создаётся коммит и выдаёт сообщение

On branch master, nothing to commit, working tree clean

Нужно добавить все файлы в коммит

git add .

После добавляем сам коммит

git commit -m "имя-коммита"
